# التصميم التقني: بوصلة القبلة (Qibla Direction)

ميزة بوصلة القبلة تبدو بسيطة للمستخدم، لكنها برمجياً تعتبر من أكثر الميزات التي تتطلب دقة هندسية لأنها تتعامل مع "مستشعرات الجهاز" (Sensors) و"الصلاحيات" (Permissions) و"الرياضيات المعقدة".

إليك الخطة التقنية الأقوى والأكثر استقراراً لبناء بوصلة قبلة احترافية (لا تهتز بشدة، وتعمل بدقة).

---

## 1️⃣ كيف تعمل البوصلة رياضياً؟ (The Logic)
لكي يشير السهم إلى الكعبة، نحتاج إلى معرفة زاويتين:
1. **اتجاه الهاتف (Device Heading):** أين يشير رأس الهاتف الآن بالنسبة للشمال الجغرافي؟ (نحصل عليه من حساس المغناطيسية في الهاتف).
2. **زاوية القبلة (Qibla Bearing):** أين تقع مكة بالنسبة لموقع المستخدم الحالي؟ (نحسبها عبر خطوط الطول والعرض).

**المعادلة النهائية:**
`دوران الإبرة في الشاشة = زاوية القبلة - اتجاه الهاتف`

---

## 2️⃣ المكتبات المطلوبة (The Packages)
بدلاً من استخدام حزم جاهزة قديمة أو غير مدعومة، سنبنيها نحن بدقة باستخدام:
1. **`geolocator`**: لجلب موقع المستخدم الحالي (خط الطول والعرض).
2. **`flutter_compass`**: لقراءة حساس المغناطيسية (اتجاه الهاتف).
3. **`adhan`**: *(مفاجأة سارة!)* بما أنك اخترت هذه المكتبة لحساب مواقيت الصلاة في خطتك الأساسية، فهي تحتوي بداخلها على دالة ممتازة لحساب القبلة بدقة: `Qibla(coordinates).direction`.

---

## 3️⃣ تحديات برمجية وحلولها (Challenges & Solutions)

### 🔴 التحدي الأول: اهتزاز إبرة البوصلة (Jittering)
حساس المغناطيسية يعطي قراءات سريعة جداً ومتذبذبة (مثلاً: 45، ثم 47، ثم 44). إذا ربطنا الإبرة بالحساس مباشرة، ستظهر الإبرة وهي "ترتجف" بشكل مزعج جداً للمستخدم.
**✅ الحل التقني:**
استخدام ودجت `AnimatedRotation` في Flutter، أو تمرير القراءات عبر (Low-pass filter)، هذا سيجعل الإبرة تتحرك بنعومة وسلاسة (Smooth Transition) كالتي في أجهزة أبل.

### 🔴 التحدي الثاني: الهواتف التي لا تحتوي على حساس (No Magnetometer)
بعض هواتف الأندرويد الاقتصادية لا تحتوي على حساس البوصلة. 
**✅ الحل التقني:**
قبل تشغيل الشاشة، يجب فحص وجود الحساس. إذا لم يكن موجوداً، نظهر رسالة تفيد بذلك، والحل البديل هو إظهار (خريطة Google Maps) يرسم عليها خط مستقيم من موقع المستخدم إلى الكعبة.

### 🔴 التحدي الثالث: التشويش المغناطيسي والتصريح
يجب التأكد من طلب صلاحية الموقع `Location Permission`، ويجب مراقبة (دقة الحساس `accuracy`). إذا كانت الدقة منخفضة، نظهر أيقونة أو تنبيه للمستخدم بعمل "حركة رقم 8" (Figure 8) لمعايرة البوصلة.

## 4️⃣ هيكل الحالة وإدارة البيانات (Riverpod State)

سنستخدم Riverpod لربط هذه البيانات الحية وتحديث الواجهة:

```dart
// 1. جلب موقع المستخدم
final userLocationProvider = FutureProvider<Position>((ref) async {
  // كود التحقق من الصلاحيات وجلب الموقع عبر Geolocator
});

// 2. حساب زاوية القبلة الثابتة بناءً على الموقع
final qiblaDirectionProvider = Provider<double>((ref) {
  final location = ref.watch(userLocationProvider).value;
  if (location == null) return 0.0;
  
  // استخدام مكتبة adhan لحساب الزاوية بدقة
  final coordinates = Coordinates(location.latitude, location.longitude);
  return Qibla(coordinates).direction;
});

// 3. مراقبة حركة الهاتف الحية
final compassHeadingProvider = StreamProvider<CompassEvent>((ref) {
  return FlutterCompass.events!;
});
```

## 5️⃣ تصميم واجهة المستخدم (UI Implementation)

لبناء شكل جميل، نحتاج لصورتين (صور شفافة PNG أو SVG):
- `compass_dial.png` (قرص البوصلة الخارجي، يحتوي على الاتجاهات N, S, E, W).
- `compass_needle.png` (إبرة البوصلة، ويفضل أن يكون رأسها بشكل الكعبة المشرفة).

```dart
Widget build(BuildContext context) {
  final qiblaDir = ref.watch(qiblaDirectionProvider);
  final compassEvent = ref.watch(compassHeadingProvider);

  return compassEvent.when(
    data: (event) {
      double heading = event.heading ?? 0;
      // حساب الدوران (بالراديان ليتوافق مع Transform)
      double qiblaAngle = (qiblaDir - heading) * (pi / 180);

      return Stack(
        alignment: Alignment.center,
        children: [
          // 1. قرص البوصلة (يلف عكس اتجاه الهاتف ليبقى الشمال في مكانه الحقيقي)
          AnimatedRotation(
            turns: -heading / 360, 
            duration: const Duration(milliseconds: 300),
            child: Image.asset('assets/images/compass_dial.png'),
          ),
          
          // 2. إبرة القبلة (تلف لتشير دائماً للقبلة بناءً على المعادلة)
          AnimatedRotation(
            turns: qiblaAngle / (2 * pi), 
            duration: const Duration(milliseconds: 300),
            curve: Curves.easeOut, // حركة ناعمة جداً تمنع الاهتزاز
            child: Image.asset('assets/images/compass_needle.png'),
          ),
        ],
      );
    },
    loading: () => const CircularProgressIndicator(),
    error: (e, _) => const Text('فشل في قراءة البوصلة'),
  );
}
```

### 💡 ملخص التخطيط لميزة القبلة:
1. لن نعتمد على حزم معلبة رديئة، بل سنصنع المنطق الخاص بنا باستخدام `adhan` و `flutter_compass` لضمان الاحترافية.
2. حلينا مشكلة اهتزاز الإبرة باستخدام `AnimatedRotation` مع مدة تحريك قصيرة.
3. فصلنا "قرص البوصلة" عن "إبرة الكعبة" في الواجهة، ليعطي تأثيراً واقعياً رائعاً للمستخدم أثناء دوران الهاتف.
