# التصميم التقني: الترجمة والقرآن كلمة بكلمة (Word-by-Word & Translations)

تعتبر هذه الميزة من أكثر الميزات المطلوبة للمسلمين غير الناطقين بالعربية، وحتى للناطقين بها لفهم معاني المفردات. لكن من الناحية البرمجية والهندسية في (Flutter)، بناء هذه الميزة يطرح "تحدياً جوهرياً" فيما يخص طريقة العرض.

---

## 1️⃣ التحدي الهندسي الجوهري (طريقة العرض)
هناك تعارض بين هدفين:
1. **المحافظة على شكل المصحف:** رسم الصفحة لتكون مطابقة تماماً لمصحف المدينة (وهو ما تقوم به حزمة `qcf_quran_plus`). هذه الحزمة ترسم الآيات ككتلة واحدة مبرمجة للحفاظ على تنسيق الأسطر وتبرير الحواف (Justification).
2. **الترجمة كلمة بكلمة:** تتطلب أن نكسر هذه الكتلة، ونضع تحت كل كلمة عربية مساحة فارغة لنكتب فيها الترجمة، وهذا سيدمر الشكل الهندسي لصفحة "مصحف المدينة" الورقي.

**🏆 الحل المعماري (نظام الوضعين - Dual Modes):**
تطبيقات مثل (Quran.com و Quran Majeed) حلت هذه المشكلة بتوفير وضعين للقراءة يختار المستخدم بينهما:
- **وضع المصحف (Mushaf Mode):** عرض الصفحات التقليدية (باستخدام `qcf_quran_plus`). هنا، ميزة كلمة بكلمة تعمل عبر **الضغط المطول** على الكلمة لتظهر الترجمة في (Tooltip/Popup) فوقها.
- **وضع التلاوة والترجمة (Translation Mode):** بدلاً من صفحات التمرير الأفقي، يكون العرض عبارة عن تمرير عمودي (Scrollable List). في هذا الوضع، الآيات لا تلتزم بشكل الصفحة الورقية، بل تُبنى الكلمات برمجياً (Widgets) وتُوضع الترجمة تحتها مباشرة.

---

## 2️⃣ هندسة وضع الترجمة (Translation Mode) في Flutter

لبناء شكل "كلمة بكلمة"، سنستخدم واجهة مكونة من تخطيط مرن `Wrap`، لأنه يسمح بانتقال الكلمات للسطر التالي تلقائياً.

```dart
// تخيل هندسي مبسط للواجهة (UI)
Wrap(
  direction: Axis.horizontal,
  alignment: WrapAlignment.start,
  textDirection: TextDirection.rtl,
  children: words.map((word) => Column(
    mainAxisSize: MainAxisSize.min,
    children: [
      Text(word.arabicText, style: UthmaniFont), // الكلمة القرآنية
      Text(word.translatedText, style: SmallFont), // الترجمة تحتها
    ],
  )).toList(),
)
```

## 3️⃣ مصدر البيانات (Data Source & Offline Strategy)

نظراً لأن تحميل بيانات "كلمة بكلمة" لكل صفحة عبر الإنترنت سيبطئ التطبيق، سنعتمد على استراتيجية الأوفلاين.

**أفضل مصدر للبيانات:** واجهة `Quran.com API v4`.
توفر هذه الواجهة معلومات مذهلة لكل كلمة:
- `text_uthmani` (النص العثماني).
- `translation` (الترجمة بأي لغة).
- `audio_url` (مقطع صوتي لنطق الكلمة فقط!).
- `page_number` و `line_number` (رقم الصفحة والسطر).

**استراتيجية التخزين (Data Strategy):**
1. سنقوم مسبقاً بتحميل قاعدة بيانات `SQLite` جاهزة (Dump) تحتوي على جميع كلمات القرآن (حوالي 77 ألف كلمة) مع ترجماتها الأساسية (عربي/إنجليزي/أردو) ونرفقها مع التطبيق (Bundled Assets).
2. بالنسبة للتراجم الكاملة للآيات (Tafsir/Translations)، سيتم توفير شاشة "إدارة التنزيلات"، يختار منها المستخدم اللغة التي يريدها، ليقوم التطبيق بتحميل ملف `JSON` أو `SQLite` للترجمة المطلوبة لتعمل أوفلاين عبر `Drift`.

## 4️⃣ هيكل البيانات المقترح (Data Models)

```dart
// نموذج الكلمة
class QuranWord {
  final int id;
  final int ayahId;
  final int surahId;
  final String uthmaniText;
  final String? translation; // يتغير حسب لغة التطبيق
  final String? audioUrl; // لنطق الكلمة المحددة
  final int lineNumber; // مفيد جداً إذا أردنا محاكاة شكل المصحف برمجياً
}

// نموذج الآية في وضع الترجمة
class AyahWithTranslation {
  final int id;
  final List<QuranWord> words;
  final String fullTranslation; // الترجمة الكاملة للآية
}
```

## 5️⃣ ميزة النطق بالكلمة (Word Audio) - ميزة تنافسية 🚀
بما أن الـ API يوفر ملفاً صوتياً (mp3) لكل كلمة بحد ذاتها، سنضيف ميزة رائعة:
عندما يقوم المستخدم بلمس (Tap) أي كلمة عربية في التطبيق، يتم فوراً تشغيل نطق هذه الكلمة باستخدام `just_audio`. 
- **التنفيذ:** نظراً لأن مقاطع الكلمات قصيرة جداً (أقل من ثانية)، سيتم تكييشها (Caching) فوراً عند أول نقرة أو عبر تحميل حزمة الصوتيات مسبقاً، مما يعطي تجربة تعلم ممتازة (خاصة لمن يتعلم القراءة الصحيحة).

---

### 💡 ملخص تخطيط الميزة:
1. لن نفسد شكل المصحف الورقي، بل سنوفر **"وضعين للعرض"** (وضع المصحف / وضع الترجمة).
2. سنستخدم **الـ Wrap Widget** لرسم الكلمات عمودياً مع ترجماتها.
3. سنعتمد على **قاعدة بيانات SQLite** مدمجة مع التطبيق لتعمل الميزة بدون إنترنت (Offline First).
4. إضافة **نطق الكلمة عند لمسها** لتكون أداة قوية لتعلم القراءة والتجويد.
