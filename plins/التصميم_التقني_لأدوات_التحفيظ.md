# التصميم التقني: أدوات التحفيظ وتكرار الآيات (Memorization Tools)

أداة التحفيظ هي واحدة من أكثر الميزات تعقيداً في تطبيقات القرآن من حيث التعامل مع الصوت (Audio Processing) وإدارة الحالة. في تقنيات (Flutter) ومع استخدام مكتبة `just_audio`، هناك عدة طرق لتنفيذها، ولكننا سنعتمد **الطريقة الهندسية الأفضل والأقل عرضة للأخطاء (Bug-free)**.

---

## 1️⃣ التحديات في التكرار الصوتي
المستخدم عادة يريد 3 إعدادات رئيسية للتحفيظ:
1. **تكرار الآية (Ayah Repeat):** تكرار كل آية عدد (X) من المرات.
2. **تكرار المقطع (Range Repeat):** تكرار نطاق الآيات (مثلاً من الآية 1 إلى 5) عدد (Y) من المرات.
3. **فاصل زمني (Delay):** التوقف لمدة قصيرة بعد كل آية ليتمكن المستخدم من القراءة خلف القارئ.

**الطريقة الخاطئة (المعقدة):** 
مراقبة الـ Stream الخاص بالمشغل (Player)، وعند انتهاء الآية، نقوم بعمل (Seek to Zero) لإعادة تشغيلها. هذه الطريقة تسبب تقطيعاً (Gaps) في الصوت، وتكون غير مستقرة عند قفل الشاشة بسبب قيود أنظمة (iOS/Android).

**الطريقة الصحيحة (التي سنعتمدها - Flattening the Playlist):**
تعتمد `just_audio` على نظام `ConcatenatingAudioSource` (قائمة تشغيل متصلة).
بدلاً من محاولة "برمجة" التكرار باستخدام الشروط (if/else)، سنقوم **ببناء وتسطيح قائمة التشغيل (Playlist) مقدماً** وإعطائها للمشغل ليقوم بتشغيلها بشكل متتالٍ وسلس (Gapless Playback).

---

## 2️⃣ آلية بناء قائمة التشغيل (Playlist Algorithm)

لنفترض أن المستخدم اختار:
- النطاق: الفاتحة (الآية 1 إلى 3)
- تكرار الآية: مرتين (2x)
- تكرار المقطع: مرتين (2x)

سيقوم `Riverpod` بتمرير هذه الإعدادات إلى طبقة الـ Service، والتي ستقوم ببناء الـ `ConcatenatingAudioSource` كالتالي:

```dart
// الشرح النظري للقائمة التي سيتم إنشاؤها:
[
  // --- التكرار الأول للمقطع ---
  AudioSource(Ayah 1), AudioSource(Ayah 1), 
  AudioSource(Ayah 2), AudioSource(Ayah 2), 
  AudioSource(Ayah 3), AudioSource(Ayah 3),
  
  // --- التكرار الثاني للمقطع ---
  AudioSource(Ayah 1), AudioSource(Ayah 1), 
  AudioSource(Ayah 2), AudioSource(Ayah 2), 
  AudioSource(Ayah 3), AudioSource(Ayah 3),
]
```
هذه الطريقة تضمن أن المشغل (Player) يتعامل معها كملفات متتالية عادية، مما يدعم التشغيل في الخلفية عبر `audio_service` بشكل مثالي دون أي تدخل برمجي معقد أثناء التشغيل.

## 3️⃣ كيفية حل مشكلة الفاصل الزمني (Delay/Pause)
إذا أراد المستخدم فاصلاً زمنياً (مثلاً: 3 ثوانٍ) بين الآيات ليقرأها هو:
لا نستخدم دالة `Future.delayed` أو إيقاف المشغل (Pause)، بل نستخدم ملف صوتي صامت (Silence) وندرجه داخل قائمة التشغيل.

```dart
// في ملفات التطبيق (Assets) نضع ملف صوتي صامت مدته ثانية واحدة (silence_1s.mp3)
// نستخدم ClippingAudioSource لتكراره أو قصه ليناسب مدة التوقف المطلوبة.

final silenceSource = ClippingAudioSource(
  child: AudioSource.uri(Uri.parse('asset:///silence.mp3')),
  end: Duration(seconds: userSelectedDelay),
);

// تصبح القائمة:
[
  AudioSource(Ayah 1), silenceSource, AudioSource(Ayah 1), silenceSource,
  AudioSource(Ayah 2), silenceSource, ...
]
```

## 4️⃣ هيكل البيانات (Data Models) وحالة التطبيق (State)

سنحتاج لإنشاء نموذج يمثل إعدادات التحفيظ:

```dart
class MemorizationSettings {
  final int startAyahId;
  final int endAyahId;
  final int ayahRepeatCount;
  final int rangeRepeatCount;
  final int delayInSeconds;
  
  // Constructor & CopyWith ...

```

في طبقة الـ **Application (Riverpod):**
سنقوم بإنشاء `StateNotifier` أو `Notifier` لإدارة هذه الإعدادات:

```dart
final memorizationSettingsProvider = NotifierProvider<MemorizationNotifier, MemorizationSettings>(() {
  return MemorizationNotifier();
});

class MemorizationNotifier extends Notifier<MemorizationSettings> {
  @override
  MemorizationSettings build() {
    return MemorizationSettings.defaultSettings();
  }
  
  void updateRange(int start, int end) { ... }
  void updateRepeatCount(int ayahRepeat, int rangeRepeat) { ... }
  
  // الدالة الأهم: تحويل هذه الإعدادات إلى Playlist وإرسالها للمشغل
  Future<void> startMemorizationSession() async {
    final settings = state;
    final audioSources = _buildPlaylist(settings);
    final playlist = ConcatenatingAudioSource(children: audioSources);
    
    await ref.read(audioPlayerProvider).setAudioSource(playlist);
    await ref.read(audioPlayerProvider).play();
  }

```

## 5️⃣ واجهة المستخدم (UI Flow) لتفعيل التحفيظ
1. **أيقونة التحفيظ:** تضاف في شريط الأدوات العلوي للمصحف أو عند الضغط المطول على آية.
2. **نافذة منبثقة (Bottom Sheet):** تظهر للمستخدم تحتوي على:
   - من آية [ X ] إلى آية [ Y ].
   - تكرار الآية: [ - 1 + ] مرات.
   - تكرار المقطع: [ - 1 + ] مرات.
   - فاصل زمني للترديد: [ لا يوجد / 3 ثوانٍ / نفس مدة الآية ].
3. **زر (ابدأ الحفظ):** يستدعي دالة `startMemorizationSession` ويظهر شريط تشغيل سفلي يوضح حالة التكرار.
